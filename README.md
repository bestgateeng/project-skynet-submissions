# Project Skynet Submissions

Welcome to the official submission hub for the Bestgate AI Hackathon.

This repository is the single source of truth for hackathon entries. Each team should create one submission issue here and use that issue to link to everything reviewers need.

## What Goes Here

Each project submission should have:

- one GitHub issue in this repository
- a short write-up
- links to the project artifacts

Your actual project does not need to live in this repository.

You can submit any of the following, as long as reviewers can access them:

- a GitHub repository
- a hosted web app
- a static HTML site
- a notebook or data workflow
- a demo video
- a Google Drive folder for large assets
- any combination of the above

## Submission Deadline

All submissions must be posted by:

**May 31 at 11:59 PM ET**

## How To Submit

1. Open a new issue in this repository.
2. Use the submission template.
3. Include the required information listed below.
4. Make sure every linked artifact is accessible to reviewers before you submit.

## Required Information

Each submission issue should include:

- **Project Name**
- **Name / Name of Team Members**
- **Short Summary**
  Briefly explain what the project is and why it matters.
- **Primary Repo or Artifact Link**
  The main link reviewers should start with.
- **Live URL**
  Include if the project is hosted.
- **Demo Video**
  Include if available.
- **Drive Folder**
  Include if you have large files, screenshots, slides, or assets.
- **Quick Start / Review Instructions**
  Explain how to run it, view it, or evaluate it.
- **AI Tools Used**
  Example: ChatGPT, Claude, Gemini, Copilot, Cursor, etc.
- **How AI Was Used**
  Explain how AI materially changed the build process.
- **Known Constraints**
  Mention any setup requirements, access requirements, or limitations.

## Write-Up Guidance

Keep the write-up to **one page max**.

A strong write-up includes:

- **The Pitch**: what the project is and what it does
- **The Links**: repo, app, video, or folder
- **The Explainer**: how AI was used to build it

## Submission Rules

- Submit **one issue per project**
- Teams should not open multiple issues for the same project
- Update your original issue if you need to fix or improve your submission
- Make sure links work and permissions are set correctly
- Do not upload large binaries directly to GitHub if a linked Drive folder is a better fit

## Reviewer Expectations

Reviewers should be able to:

- understand the project quickly
- access the main artifact without guesswork
- see how AI was used
- run, inspect, or evaluate the submission with reasonable effort
